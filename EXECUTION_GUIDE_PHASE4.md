## 🏃 Execution: Step 2 - Production Evaluation & Promotion

### Objective
Evaluate the candidate model on held-out test data and compare against current champion. Promote challenger if it outperforms the champion.

### Steps in Databricks

**1. Open Notebook**: `08_Production.py`

**2. Verify Production Configuration**:
```python
# Top of notebook should show:
production_run_name = "production_evaluation"
selection_metric = "auc_pr"  # Decision metric
uc_model_name = "workspace.ana_martin17.manufacturing_quality_control_model"
baseline_table_name = "workspace.ana_martin17.gold_inspection_test_baseline"
```

**3. Run Cell: "Initialize Production Pipeline"**
```python
# Expected output:
# Production evaluation started at: YYYY-MM-DD HH:MM:SS UTC
# Candidate version: N
# New production run ID: <run_id>
# No previous champion exists (cold start)
```

**4. Run Cell: "Retrain Candidate as Challenger on Train+Validation"**
- Loads candidate hyperparameters
- Retrains on training + validation datasets (18 months total)
- Logs as: `challenger_model` artifact
- **Expected runtime**: ~15-30 minutes

**5. Run Cell: "Evaluate Challenger on Test Set"**
- Loads challenger model
- Evaluates on held-out test data (6 months)
- Logs metrics: auc_pr, auc_roc, f1, recall, precision, accuracy
- Generates diagnostic figures: PR curve, ROC curve, confusion matrix, calibration curve
```python
# Expected output:
# Challenger test AUC-PR: 0.8XXX
# Challenger test AUC-ROC: 0.9XXX
# Challenger test F1-score: 0.7XXX
```

**6. Run Cell: "Promotion Decision Logic"**
- Compares challenger test AUC-PR against champion (if champion exists)
- For **cold start** (no champion): challenger promoted automatically
- For **warm start**: challenger must strictly exceed champion
```python
# Expected output (first run):
# Decision: CHALLENGER PROMOTED
# Reason: No champion exists. Challenger promoted directly.
```

**7. Run Cell: "Full Refit & Registration as Champion"**
- Retrains on ALL data (training + validation + test)
- Registers as new version
- Assigns `champion` alias
- Updates metadata tags with test metrics
- **Expected runtime**: ~15-30 minutes

**8. Run Cell: "Write Test Baseline for Monitoring"**
- Transforms test set with champion model
- Extracts probability predictions
- Writes to: `gold_inspection_test_baseline` in Unity Catalog
- Adds monitoring metadata tags
```python
# Expected output:
# Test baseline written to workspace.ana_martin17.gold_inspection_test_baseline
# Rows: ~5,000,000 (test set size)
# Tags applied: production_run_id, champion_version, test_auc_pr, etc.
```

**9. Run Cell: "Pipeline Completion & Artifact Cleanup"**
```python
# Expected output:
# Production evaluation completed successfully
# New champion version: N
# Model URI: workspace.ana_martin17.manufacturing_quality_control_model/champion
# Temporary artifacts cleaned up
```
